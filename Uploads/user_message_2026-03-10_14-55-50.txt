# PROMPT — Refatoração Visual Completa do App Excel

## Contexto
Você vai refatorar a identidade visual de um aplicativo mobile de aprendizado de Excel (estilo Duolingo). O objetivo é substituir a paleta de cores inconsistente atual por um sistema coeso baseado em verde, e modernizar o HTML das descrições de questões seguindo um design system padronizado.

## PARTE 1 — Nova Paleta de Cores (CSS Variables)

Substitua TODAS as cores hardcoded do projeto pelas variáveis abaixo. Declare-as no `:root` global do CSS (globals.css, theme.ts, tailwind.config, ou onde o projeto centraliza tokens de design):

```css
:root {
  /* Primárias */
  --color-primary:        #217346;  /* Verde Excel — botões principais, headers, CTA */
  --color-primary-mid:    #2E9E5B;  /* Verde médio — hover, gradientes, progresso */
  --color-primary-light:  #E8F5E9;  /* Verde suave — fundos de card, highlight ativo */
  --color-primary-vivid:  #27AE60;  /* Verde vivo — XP ganho, badges de sucesso */

  /* Neutros */
  --color-bg:             #F7F9F7;  /* Off-white — fundo geral das telas */
  --color-surface:        #FFFFFF;  /* Branco puro — cards, modais */
  --color-text-main:      #1A1A2E;  /* Quase preto — texto principal */
  --color-text-sub:       #4A4A6A;  /* Cinza azulado — texto secundário, labels */
  --color-border:         #E0E5E0;  /* Cinza claro — bordas, divisores */
  --color-gray:           #B0BEC5;  /* Cinza médio — estados bloqueados, placeholders */
  --color-gray-bg:        #F0F2F0;  /* Cinza fundo — cards bloqueados */

  /* Semânticas */
  --color-amber:          #F59E0B;  /* Âmbar — PRO, premium, avisos */
  --color-amber-light:    #FEF3C7;  /* Âmbar claro — fundo de dicas/hints */
  --color-red:            #E74C3C;  /* Vermelho — erros, resposta errada */
  --color-red-light:      #FEE2E2;  /* Vermelho claro — fundo de erro */
}
```

### Regras de uso obrigatórias:
- **Verde** (`--color-primary`, `--color-primary-mid`): headers, botões de ação, barras de progresso, ícones ativos, navegação ativa
- **Verde claro** (`--color-primary-light`): fundo de opção selecionada, fundo de resposta correta, highlight de célula Excel
- **Cinza** (`--color-gray`, `--color-gray-bg`): trilhas bloqueadas, estados inativos, separadores
- **Âmbar** (`--color-amber`): badge PRO/Premium, streaks, dicas, avisos
- **Vermelho** (`--color-red`, `--color-red-light`): resposta errada, alertas críticos
- **PROIBIDO**: roxo, laranja, marrom, rosa, azul e qualquer outra cor decorativa nas trilhas

Agora aplique os padrões acima reescrevendo o campo `description` de TODAS as questões do seed (`seed_trails.ts`), seguindo o mapeamento abaixo:

| Questão | Tipo HTML a usar |
|---|---|

### Regras obrigatórias ao reescrever:
1. **NUNCA** use `style=""` inline com cores fora da paleta (`#2196F3`, `#667eea`, `#764ba2`, `#FF5722`, `#4CAF50` genérico, etc.)

3. **REMOVA** todos os `<div class="excel-header">`, `<div class="excel-row">`, `<div class="cell">` — substitua pela estrutura `<table class="xl-grid">`
4. **REMOVA** emojis usados como ícones decorativos nos headers de card — use apenas nas `ctx-label` quando reforçar o significado
5. **MANTENHA** o texto informativo de cada questão, apenas atualize o container visual
